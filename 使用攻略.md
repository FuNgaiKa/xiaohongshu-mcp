# 小红书 MCP Windows 使用攻略

## 1. 下载

从 Release 页面下载 Windows 版压缩包，解压后包含两个文件：

```
xiaohongshu-login-windows-amd64.exe   # 登录工具
xiaohongshu-mcp-windows-amd64.exe    # MCP 服务
```

---

## 2. 首次登录（获取 Cookie）

> 必须用**管理员身份**运行 cmd，否则 Chromium 解压时会报 `Access is denied`。

1. 按 `Win` 键，搜索 `cmd`，右键 → **以管理员身份运行**
2. 执行登录工具：

```cmd
"C:\Users\你的用户名\Downloads\xiaohongshu-mcp-windows-amd64\xiaohongshu-login-windows-amd64.exe"
```

3. 首次运行会自动下载 Chromium（约 100MB），耐心等待
4. 浏览器弹出后，手动扫码/输入账号密码登录小红书
5. 登录成功后程序自动保存 `cookies.json` 并退出

> **注意**：`cookies.json` 保存在 exe 所在目录，下次运行 MCP 服务时会自动读取。

---

## 3. 启动 MCP 服务

登录成功后，运行 MCP 服务（普通 cmd 即可）：

```cmd
"C:\Users\你的用户名\Downloads\xiaohongshu-mcp-windows-amd64\xiaohongshu-mcp-windows-amd64.exe"
```

服务默认监听：`http://localhost:18060/mcp`

---

## 4. 配置 MCP 客户端

在支持 MCP 的客户端（Claude Code、Cursor 等）中配置：

```json
{
  "mcpServers": {
    "xiaohongshu-mcp": {
      "type": "http",
      "url": "http://localhost:18060/mcp"
    }
  }
}
```

Claude Code 的配置文件路径：`.mcp.json`（项目根目录）

---

## 5. 常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `Access is denied` rename 失败 | 权限不足 | 用管理员身份运行 cmd |
| `failed to load cookies` 警告 | 第一次运行没有缓存 | 正常，忽略即可 |
| 运行 darwin-arm64 文件报错 | 该文件是 macOS 版本 | 下载 windows-amd64 版本 |
| `此时不应有 &` | cmd 不支持 `&` 语法 | 去掉 `&`，直接运行路径 |

---

## 6. 重新登录

Cookie 过期后，重新以管理员身份运行登录工具即可，Chromium 已下载无需再次下载。
